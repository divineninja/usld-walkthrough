# Agent Walkthrough Guide

This guide provides a step-by-step walkthrough of the agent workflow for registering listing information with two payment process options: 1) Agent registers the customer's credit card, and 2) Agent sends a payment link to the customer for self-registration. The walkthrough is based on the screenshots in the `Agent` directory.

[Agent Login Page](https://dashboard.uslocaldirectory.com/agent)

## Workflow Summary

This agent workflow demonstrates the complete process for registering listing information with flexible payment options:

1. **Login** - Agent authenticates into the system
2. **Dashboard** - Access main interface and start registration
3. **Registration** - Enter listing details, schedule, and customer info
4. **Summary** - Review all entered information
5. **Payment Choice** - Select between direct card registration or payment link
6. **Payment Processing** - Complete the chosen payment method
7. **Confirmation** - Verify successful registration and listing status

## Payment Options Overview

- **Payment Link Option:** Allows customers to register their own payment details securely via a link, reducing agent handling of sensitive information.
- **Direct Card Registration:** Enables agents to register customer credit cards directly when preferred or necessary.

## Notes

- All screenshots were captured sequentially to demonstrate the agent workflow.
- The workflow supports both payment methods, providing flexibility for different customer scenarios.
- Security and compliance should be prioritized, especially when handling payment information.

## For Developers

When implementing or modifying this workflow:
- Ensure secure handling of payment information
- Implement proper validation for all form fields
- Provide clear user feedback at each step
- Support both payment options seamlessly
- Include error handling and recovery mechanisms
- Maintain audit trails for registration activities
